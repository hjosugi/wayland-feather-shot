# セキュリティ / プライバシー

wayland-feather-shotはローカル専用です。

## 設計による削除

- クラウドアップロード（Imgurクライアントなし、アップロードボタンなし）
- アカウントログイン
- テレメトリ / クラッシュレポート
- ネットワーク同期またはバックグラウンドネットワークタスク
- いかなる種類のネットワークコード — 無効にするものはありません

## ローカルアクションのみ

- xdg-desktop-portalによって返されたスクリーンショットファイルのURIを読み取り、読み込んだ後にポータルの一時ファイルを削除します。
- ユーザーが選択したローカルディレクトリにPNG/JPEG/WebPを保存します。
- レンダリングされた画像をローカルWaylandクリップボードにコピーします（利用可能な場合は`wl-copy`を介して、そうでない場合はバンドルされたローカルホルダープロセス、さらにそうでない場合はプロセス内のGDKクリップボードを介して）。ホルダーはクリップボードのみを所有し、一時PNG以外のファイルを開かず、ソケットも開きません。
- Ctrl+PrtSc用のオプションのGlobalShortcutsポータルセッション。
- 明示的なユーザーアクションでのみ開始され、完了時に閉じられるオプションのScreenCastポータル + PipeWireセッションによるスクロールキャプチャ。

## Waylandセキュリティモデル

アプリは決してコンポジタをバイパスしません。すべての画面アクセスはXDGデスクトップポータルのリクエスト/許可フローを通じて行われます。スクロールキャプチャは表示されているフレームのみを読み取り、他のウィンドウのオフスクリーンコンテンツを読み取ったり、入力を注入したりすることはできません。
